# **前端設計規格文件 (Frontend Design Spec) | 智慧產業數據管道與 AI 自動化驗證系統 MVP**

## **1\. 文件基本資訊**

* **專案名稱：** 智慧產業數據管道與 AI 自動化驗證系統 (Smart-Data Pipeline & AI Harness System)  
* **文件版本：** v1.0  
* **目標：** 指引 Stitch / AI 快速生成高保真度 (High-fidelity) 的單頁式 (SPA) 互動儀表板 MVP。  
* **主要展示重點：**  
  1. 數據管道（ETL & SQL）的即時流向。  
  2. **裝具工程 (Harness) 驗證過程：** 包含 AI 幻覺攔截、自動重試修正的動態動畫。  
  3. **三層邊界控制 (Boundary)：** 「Ask (人機協同確認)」的互動彈窗。

## **2\. UI/UX 設計風格與佈局 (Design Style & Layout)**

為了符合大台中在地精密製造業（工廠端）或智慧醫療（護理站）的場景，我們採用 **「暗色系現代工業風格 (Dark Industrial Dashboard)」**，這能提供極佳的**可讀性 (Scannability)**，並讓數據警示與 AI 狀態更加醒目。

### **2.1 配色與視覺指標**

* **背景主色 (Background)：** 炭灰色 / 深藍黑 (\#0B0F19 或 \#121826)，展現專業系統質感。  
* **主要狀態色：**  
  * **Success / Pass (通過)：** 翡翠綠 (\#10B981) ![][image1] 代表數據清洗成功、Harness 驗證通過。  
  * **Warning / Ask (詢問)：** 琥珀黃 (\#F59E0B) ![][image1] 代表異常分數過高，觸發人機確認機制。  
  * **Error / Intercept (攔截)：** 烈焰紅 (\#EF4444) ![][image1] 代表 AI 格式錯誤、或偵測到隱私洩漏被系統安全攔截。  
  * **AI Processing (處理中)：** 科技藍 (\#3B82F6) ![][image1] 代表正在呼叫 Claude Sonnet。

### **3.1 元件 A：數據管道監控（Live Pipeline Step）**

* **展示目的：** 證明你具備整合 Python \+ SQL \+ AI 管道的架構能力。  
* **互動設計：**  
  * 設計三個狀態節點（Node）：Ingestion ![][image1] ETL & SQL ![][image1] AI & Harness。  
  * 當用戶點擊「模擬控制台」的「發送新數據」時，這三個節點會依序亮起「處理中」的藍色光暈，最終變為代表成功的綠色。

### **3.2 元件 B：裝具工程防護罩（Harness Guardrail Log）- *面試吸睛核心！***

* **展示目的：** 展示你是如何「控制 AI 產出品質」，而非盲目相信 Prompt。  
* **版面設計：**  
  * **左側 (Raw & Output)：** 顯示模擬的原始 Log（如設備異常溫度）與 AI 回傳的 JSON 原始碼。  
  * **右下 (Harness Run Log)：** 滾動式的代碼風格日誌。  
  * **模擬情境按鈕 \- 「注入錯誤/幻覺數據」：**  
    * 點擊後，前端會模擬發送一份「不完整/格式混亂的 AI 回傳」。  
    * **Harness 驗證日誌**立刻閃爍紅色警示：\[WARN\] 欄位 'device\_id' 缺失。觸發自動修正重試 (Attempt 1/3)...  
    * 隨後日誌更新為綠色：\[SUCCESS\] 修正成功！回傳資料格式 100% 通過邏輯檢查。

### **3.3 元件 C：三層邊界控制台與 Ask 彈窗 (Boundary Controller)**

* **展示目的：** 展現實體工業/醫療級的安全防護思維 (Always/Ask/Never)。  
* **Always/Never 指示燈：** 畫面右側顯示三個盾牌圖示：  
  * Always Check (格式始終驗證 \- 綠色常亮)  
  * Never Outflow (敏感個資不外傳至外部 LLM \- 藍色常亮)  
* **Ask 互動彈窗 (Modal)：**  
  * 當用戶點擊「模擬高危異常數據」時，畫面中央跳出琥珀黃警示彈窗：  
    * **"⚠️ 三層邊界安全警告：偵測到機台溫度超載且異常分數達 ![][image2] 分，是否立即啟動冷卻程序並通知工廠課長？"**\[cite: 1\]  
    * 提供按鈕：\[確認執行\]（按下去後更新系統狀態為已處理）與 \[取消駁回\]。

### **3.4 元件 D：SQL 歷史紀錄表 (SQL Database Log View)**

* **展示目的：** 證明清洗完的數據有確實寫入 SQL，並能高效撈取。  
* **版面設計：**  
  * 包含時間戳記、來源、原始數據摘要、AI 解析結論。  
  * 特別加入一欄 **「Harness 修正次數」**（例如：0次、1次），用視覺再次向技術面試官強調你後端的自動驗證機制有多強大。

## **5\. 給 Stitch / AI 的提示詞工程模板 (Prompt Template for UI Generation)**

當你將這份文件餵給 Stitch 或 Claude 時，可以使用以下提示詞，引導其完美生成代碼：

**"Please act as an expert Senior Frontend Developer. Based on the attached Frontend Design Spec, generate a single-file React/Tailwind CSS or HTML/Tailwind dashboard for my MVP. It must be highly interactive. The 'Simulator Controls' at the top should let me toggle between: 1\. Normal Flow (Green, 0 retries), 2\. AI Hallucination & Harness Intercept (Shows Red error log first, then auto-resolves to Green after 1.5 seconds), 3\. Trigger High-risk Alert (Triggers the 'Ask' amber modal for human confirmation). Make the UI feel like an advanced industrial monitoring system with rich scannability, subtle animations, and clean code."**

## **6\. 前端開發步驟指引 (Sprint Actions)**

1. **Step 1:** 將本 .md 檔案複製到你的開發目錄中。  
2. **Step 2:** 將第 5 點的提示詞與本規格書輸入給 **Stitch**，生成前端原型。  
3. **Step 3:** 預覽生成的前端，測試「模擬控制台」按鈕，確保三種模擬情境（正常、裝具攔截、Ask彈窗）的動畫與流程完全順暢。  
4. **Step 4:** 後續與後端（Python ETL \+ FastAPI）整合時，直接將 Mock state 替換為實際的 fetch() 呼叫即可！
